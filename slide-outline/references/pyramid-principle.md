# The Pyramid Principle for Presentations

## Core Concept

The Pyramid Principle (Barbara Minto, McKinsey) structures communication top-down:
start with the conclusion, then support it with grouped arguments, each backed by
evidence. Applied to presentations, this means every slide exists to support the
deck's key message.

## Structure

```
Key Message (Title Slide)
├── Argument 1 (Act / Section)
│   ├── Evidence 1.1 (Slide)
│   ├── Evidence 1.2 (Slide)
│   └── Evidence 1.3 (Slide)
├── Argument 2 (Act / Section)
│   ├── Evidence 2.1 (Slide)
│   └── Evidence 2.2 (Slide)
└── Conclusion / CTA (Closing)
    └── Next Steps (Slide)
```

## Rules for Slide Titles

1. **Assertion, not label** — "Revenue grew 40% YoY" not "Revenue Update"
2. **Standalone meaning** — Reading only titles should tell the full story
3. **One message per slide** — If you need "and", split into two slides
4. **Action-oriented** — Use active verbs: "drove", "enabled", "reduced"

## The SCQA Framework for Setup

Structure the opening act using Situation-Complication-Question-Answer:

- **Situation**: What the audience already knows (shared context)
- **Complication**: What changed or went wrong (the tension)
- **Question**: The question this raises (implicit or explicit)
- **Answer**: Your key message (the rest of the deck proves this)

Example:
- S: "We launched AI analytics in Q3 2025"
- C: "But adoption stalled at 40% after the first month"
- Q: "How do we reach 90% adoption?"
- A: "Three targeted interventions drove adoption to 92%"

## Act Structure Patterns

### Problem → Solution → Impact (most common)
1. Setup: Define the problem and its cost
2. Solution: Present what was done
3. Impact: Show results and next steps

### Chronological Narrative
1. Before: Previous state
2. During: What happened / what we did
3. After: Current state and outcomes

### Recommendation with Evidence
1. Recommendation: State the ask upfront
2. Evidence: Three supporting arguments
3. Action: Specific next steps and timeline

## Slide Count Heuristics

- 1 slide per minute of presentation (average)
- Title and section divider slides need < 30 seconds
- Data slides need 90-120 seconds to explain
- Never exceed 30 slides for any presentation length
- If you have > 20 slides, look for slides to cut, not to rush

## Anti-Patterns

| Anti-Pattern | Problem | Fix |
|---|---|---|
| Agenda slide | Wastes a slide on structure the audience doesn't need | Cut it; the narrative should be self-guiding |
| "Thank you" slide | Dead end with no value | Replace with CTA or key takeaway |
| Topic-label titles | "Market Overview" tells audience nothing | Rewrite as assertion: "Market grew 12% but competition intensified" |
| Wall of bullets | Too much text, audience reads instead of listens | Max 3-5 bullets, max 10 words each |
| Data dump | Chart without interpretation | Lead with the insight, use the chart as evidence |
| Slide deck as document | Trying to be a report and a presentation | Presentations are visual aids for a speaker, not standalone documents |
